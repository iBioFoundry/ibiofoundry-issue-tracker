# ibiofoundry-issue-tracker

> Bug reports and issue tracking for the NSF iBioFoundry project

## Overview

This is the **public-facing issue tracker** for the iBioFoundry platform. It serves as the central intake point for bug reports, feature requests, and general feedback from users and collaborators.

This repository does **not** contain any source code. Its purpose is to collect, triage, and route issues to the appropriate repositories and developers within the iBioFoundry organization.

## How to Submit an Issue

1. [Open a new issue](../../issues/new/choose)
2. Choose the appropriate issue template
3. Fill in as much detail as possible, including:
   - Steps to reproduce
   - Expected vs. actual behavior
   - Screenshots or logs if available
   - Environment (browser, OS, staging vs. production)

## Triage Process

Once an issue is submitted:

1. A maintainer will review and label the issue
2. The issue will be triaged and assigned to the relevant repository within the iBioFoundry org
3. The issue will be transferred to the appropriate repo and assigned to the responsible developer
4. You'll be notified via GitHub when the issue is transferred or resolved

## Labels

| Label | Description |
|-------|-------------|
| `bug` | Something isn't working as expected |
| `feature-request` | A new feature or enhancement |
| `needs-triage` | Awaiting review and routing |
| `transferred` | Issue has been moved to a specific repo |
| `duplicate` | Already reported |
| `invalid` | Not a valid issue |

## Contact

For urgent issues or security vulnerabilities, please do not open a public issue. Contact the iBioFoundry team directly at i-CBF@illinois.edu
